### 13. **NO CIRCUIT BREAKER FOR EXTERNAL APIS** 🟠
**Files:** [src/thunai/intelligence/slm/ollama.py](ollama.py), [sarvam.py](interaction/sarvam.py)

**What Needs to Happen:**
1. Implement circuit breaker pattern for all external API calls:
   ```python
   class CircuitBreaker:
     def __init__(self, failure_threshold=5, timeout=300):
       self.failures = 0
       self.threshold = failure_threshold
       self.timeout = timeout
       self.last_failure_time = None
       self.state = 'closed'  # closed, open, half-open

     def call(self, func, *args):
       if self.state == 'open':
         if time.time() - self.last_failure_time > self.timeout:
           self.state = 'half-open'  # Try recovery
         else:
           raise CircuitBreakerOpenError()
       
       try:
         result = func(*args)
         if self.state == 'half-open':
           self.state = 'closed'  # Recovered
         return result
       except Exception as e:
         self.failures += 1
         self.last_failure_time = time.time()
         if self.failures >= self.threshold:
           self.state = 'open'
         raise
   ```
2. Apply to: Ollama, Gemini, Sarvam, ElevenLabs APIs
3. Expose circuit state via `/health/providers`

**Acceptance Criteria:**

- [ ] Tests verify state transitions

## RISK MITIGATION

### High-Risk Items (Monitor Closely)

1. **Edge Hardware Integration (RV1126)**
   - Risk: RKNN SDK compatibility, thermal throttling, BLE stability
   - Mitigation: Early prototype, test on real hardware by week 6
   - Fallback: Reduce model complexity if latency budget exceeded

2. **LLM Cost Escalation**
   - Risk: Gemini/Claude credits exhausted if quota not enforced
   - Mitigation: Monthly cost forecast, rate limits per user
   - Alert: Budget threshold monitoring

3. **Mobile Battery Drain**
   - Risk: BT polling + screen + TTS drains battery too fast
   - Mitigation: Profile on real device, optimize polling interval
   - Test: 8-hour drive without charging

4. **Database Scaling**
   - Risk: PostgreSQL cannot handle 10k+ concurrent users
   - Mitigation: Connection pooling, query optimization, read replicas
   - Monitor: Query latency p95 < 500 ms
